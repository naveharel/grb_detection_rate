# Publication figures — quick start

Static, journal-quality figures (PNG by default; vector PDF opt-in) generated by
importing the read-only physics engine in [`grb_detect/`](../grb_detect/). Separate
from the in-browser app.

**Visual standards + the mandatory inspection gate:**
[`docs/figure_standards.md`](../docs/figure_standards.md). **Build & `figlib` reference:**
[`docs/figures.md`](../docs/figures.md).

## Setup (one time)

```
.venv\Scripts\python -m pip install -r ..\requirements-figures.txt
```

Text renders through **LaTeX by default** (`text.usetex`), which needs a system TeX
toolchain (`latex` + `dvipng`, e.g. MiKTeX / TeX Live — not a pip package). With no TeX
installed, figures fall back to matplotlib mathtext (STIX) automatically. Force the
choice with `style.use_style(usetex=False)` or `GRBFIG_USETEX=0/1`.

## Run a figure

```
.venv\Scripts\python figures\fig_qmedian_vs_Fdec.py
```

Output → `figures/output/` (gitignored). PNG only by default. The vector PDF (paper
deliverable) is opt-in — pass `formats=("pdf", "png")` to `io.savefig_pub` when you
specifically need it.

## Shared library (`figlib/`)

| Module | Purpose |
|--------|---------|
| `style` | AAS/ApJ style (`styles/grbpaper_aas.mplstyle`), `use_style()`, `figsize()`, palettes, color roles, regime colors |
| `presets` | `SurveyPreset` mirror of `web/app.js` `PRESETS` (ZTF, Rubin) — keep in sync with `web/app.js` |
| `models` | `build_model_from_preset()`, `ztf_strategy_point()`, `n_exp_max()` |
| `compute` | `q_median_at()`, `medians_at()`, `rate_at()`, `regime_id_at()`, `regime_family()` |
| `overrides` | copy-on-write model surgery — vary quantities past the public API (e.g. `set_F_dec`) |
| `axes` | `set_loglog()`, `faded_guide_line()`, `draw_q_scales()`, `draw_F_dec()`, `shade_family_bands()`, `place_legend()` |
| `io` | `savefig_pub()` → writes `<name>.png` to `output/` (PDF opt-in via `formats=`) |

## Conventions in one line each

- Write a new figure with the 5-step pattern (style → model → compute → plot → save);
  scripts expose `make_figure()` and are runnable directly. See `docs/figures.md` §4.
- To vary engine internals the public API hides, use `figlib.overrides` — never edit
  the engine. See `docs/figures.md` §5 for the override contract.
- Labels are math strings `$...$` (render in both usetex and mathtext). Presets mirror
  `web/app.js`. Derive `N_exp` from the model, never hardcode `27500/47`.
- Apply color by role (`style.PRIMARY` / `ACCENT` / `REFERENCE`) and place legends with
  `axes.place_legend(ax)`; every figure must pass the visual-inspection gate before it's
  done. Full spec + checklist: `docs/figure_standards.md`.

## Example

`fig_qmedian_vs_Fdec.py` — `q_median` of ZTF-detected GRBs vs the overridden flux
normalization `F_nu,dec^(G)`: cadence-limited floor, flux-limited rise, `q_nr/√2`
ceiling. Reproduces the app's `q_med_ztf` exactly at the fiducial.
